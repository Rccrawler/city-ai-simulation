# Contributing to City AI 🤝

First off, thank you for considering contributing to City AI! It's people like you that make open-source projects such a great community to learn, inspire, and create.

## 🛠️ How Can I Contribute?

### 1. Reporting Bugs
If you find a bug in the simulation or the AI server, please open an issue. Provide as much detail as possible:
- Steps to reproduce the bug.
- Expected behavior vs. actual behavior.
- Logs or screenshots if applicable.

### 2. Suggesting Enhancements
Have an idea to improve Valle Silencio? We'd love to hear it! Open an issue to discuss it before you start coding to ensure it aligns with the project's vision.

### 3. Pull Requests
If you want to write code, that's awesome!
1. Check the `Issues` tab for tasks labeled `good first issue` or `help wanted`.
2. Fork the repository.
3. Create a new branch for your feature (`git checkout -b feature/AmazingFeature`).
4. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
5. Push to the branch (`git push origin feature/AmazingFeature`).
6. Open a Pull Request.

## 📝 Code Guidelines
- Keep the code clean and well-documented.
- If you are touching the AI prompt logic, please test it locally to ensure the NPCs don't break character.
- Try to use English for variables, commits, and pull requests.

Thank you for helping us build Valle Silencio! 🏘️
